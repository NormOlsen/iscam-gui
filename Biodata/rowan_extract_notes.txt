Hi Chris,
I've posted the catch dump files to \\DCBCPBSNA01a\SRF$\Public\PSARC\ARF14.

The one file `cat602.rda` is a binary dump of a list object with 9 potential catch data sources (data frames, also posted individually) :

1. ph3cat = PacHarv3 database (all fisheries)
2. gfcdat = GFCatch database (trawl, trap, h&l)
3. phtdat = PacHarvest database (groundfish trawl)
4. phhdat = PacHarvHL database (halibut bycatch from DMP validated landings)
5. phsdat = PacHarvSable database (sablefish trap)
6. phvdat = PacHarvHL database (validated landings Sched II & ZN)
7. phfdat = PacHarvHL database (fisherlog records Sched II & ZN)
8. fosdat = GFFOS database on the GFSH server (all fisheries)
9. gfbdat = GFbioSQL database (research and surveys)

> sapply(cat602,dim)
     ph3dat gfcdat phtdat phhdat phsdat phvdat phfdat fosdat gfbdat
[1,]    915 202276 190394  17466  54809  23483 195891 118221  45894
[2,]     12     14     13     14     11     14     14     12     13

Notes: 
a) These queries collect catch information you will likely not need: POP, rockfish other than POP (ORF), and a variety of species depending on the fishery.
b) Probably best to ignore the catches from PacHarv3 (ph3dat) and just use gfcdat.
c) Items 6 & 7 above (phvdat and phfdat) may contain redundant data. Best ask Lisa about it, because I forget.
d) Arrowtooth Flounder catches will appear in the fields marked `landed` and `discard`.
e) A quick summary on the landed catch of ARF in the first 8 databases tells me that ARF is predominantly caught by the trawl fishery. Therefore you will likely only need to look at GFCatch, PacHarvTrawl, and GFFOS. But check each data source for yourself (e.g. sum (landed and discard)).
f) The GFBio survey catches were queried later and have not been standardised to fit the catch reconstruction. However, you should be able to get annual survey catches (see last table below).
g) I've also posted output from Norm's survey query (`gfbdat.norm.rda`) in case you need catches by survey.

Hi Chris,
I've extracted biological data for Arrowtooth Flounder (ARF, 602) and posted it to \\DCBCPBSNA01a\SRF$\Public\PSARC\ARF14.

The query used to grab the data is called "gfb_bio.sql". There was a bit of post-processing in R using the function `processBio` from the R package PBStools.

Current PBS binary packages for Windows can be found at:
\\DCBCPBSNA01a\SRF$\Public\Rpackages 

Source code (tar.gz) for PBStools can be found at:
\\DCBCPBSNA01a\SRF$\Public\Rpackages\PBStools\Version 

You can get the SQL queries three ways:

1. Locate the `sql` directory in your R library tree under PBStools (if you install the package).

2. Look in the source code (tar.gz) for the directory `pkgname\PBStools\inst\sql`.

3. Get (not git) the project using SVN:
svn checkout http://pbs-tools.googlecode.com/svn/trunk/ pbs-tools-read-only
